systemctl restart 和先 stop 后 start 区别

Posted on 2020-12-08 Edited on 2020-12-12

区别

  • restart

    会把依赖该服务的其它服务重启。

  • 先 stop 后 start

    服务需手动 start 强依赖它的所有服务,
    因为 stop 服务会导致强依赖的它的服务也停止，

  • 说明

    通过 Wants= 或 WantedBy= 建立的, 是弱依赖
    通过 Requires= 或 RequiredBy= 建立的，强依赖
